# 技术选型文档

## 1. 技术选型原则

- **轻量化**: 选择轻量级技术，保证低内存占用和快速启动
- **跨平台**: 选择跨平台技术栈，支持Windows、macOS、Linux
- **现代化**: 使用现代化的技术栈和开发工具
- **性能优先**: 优先考虑性能，确保流畅的用户体验
- **可维护性**: 选择成熟稳定、易于维护的技术

## 2. 核心技术栈

### 2.1 桌面应用框架

**选择: Tauri 2.x**

**理由**:
- ✅ 轻量化：打包后体积小（<10MB），运行时内存占用低（<50MB）
- ✅ 跨平台：支持Windows、macOS、Linux三平台
- ✅ 安全性：使用系统WebView，更安全
- ✅ 性能：Rust后端性能优异，前端使用Web技术灵活
- ✅ 现代化：支持最新的Web标准和Rust特性

**替代方案考虑**:
- Electron：功能强大但体积大、内存占用高（不满足轻量化要求）
- .NET MAUI：原生性能好，但跨平台支持有限，界面现代化程度低

### 2.2 前端框架

**选择: React 18 + TypeScript**

**理由**:
- ✅ 成熟稳定：React是最流行的前端框架之一
- ✅ 类型安全：TypeScript提供类型检查，减少错误
- ✅ 生态丰富：React生态丰富，组件库和工具多
- ✅ 性能优秀：React 18的并发特性提升性能
- ✅ 易于维护：组件化开发，代码结构清晰

### 2.3 UI库

**选择: Tailwind CSS + shadcn/ui**

**理由**:
- ✅ **Tailwind CSS**: 实用优先的CSS框架，快速开发，体积小
- ✅ **shadcn/ui**: 基于Radix UI和Tailwind的现代化组件库
  - 组件质量高，设计现代化
  - 可定制性强，支持主题定制
  - 符合Apple风格设计需求

### 2.4 动画库

**选择: Framer Motion**

**理由**:
- ✅ 性能优秀：使用GPU加速，支持60fps流畅动画
- ✅ API简洁：声明式API，易于使用
- ✅ 功能强大：支持复杂动画、手势、布局动画等
- ✅ Apple风格：动画自然流畅，符合Apple风格设计

**物理引擎（可选）**:
- **matter.js**: 如果需要复杂的物理模拟（如水晃动），可使用matter.js
- 自定义物理计算：对于简单效果，可以自定义计算，性能更好

### 2.5 数据存储

**选择: SQLite（使用rusqlite或sqlx）**

**理由**:
- ✅ 轻量化：SQLite是轻量级数据库，无需单独服务器
- ✅ 本地存储：所有数据本地存储，保护隐私
- ✅ 性能优秀：对于单用户应用，SQLite性能足够
- ✅ Rust支持：rusqlite和sqlx都是成熟的Rust SQLite库
- ✅ 跨平台：SQLite跨平台支持良好

**推荐库**:
- **rusqlite**: 简单易用，适合小型应用
- **sqlx**: 编译时SQL检查，类型安全，适合大型应用

### 2.6 状态管理

**选择: Zustand**

**理由**:
- ✅ 轻量化：体积小，API简洁
- ✅ 性能优秀：基于React Context，性能好
- ✅ 易于使用：学习曲线低，代码简洁
- ✅ TypeScript支持：完整的TypeScript支持

**替代方案考虑**:
- Redux：功能强大但体积大，配置复杂（对于本应用过于复杂）
- Jotai：原子化状态管理，但本应用不需要如此细粒度

### 2.7 构建工具

**选择: Vite**

**理由**:
- ✅ 快速：开发时启动速度快，HMR快速
- ✅ 现代化：支持最新的ES特性
- ✅ 配置简单：默认配置即可使用
- ✅ Tauri集成：Tauri官方推荐使用Vite

### 2.8 打包工具

**选择: Tauri CLI**

**理由**:
- ✅ 官方工具：Tauri官方提供的打包工具
- ✅ 跨平台：支持Windows、macOS、Linux打包
- ✅ 配置灵活：支持自定义打包配置
- ✅ 安装包：生成各平台的安装包（.msi、.dmg、.deb等）

## 3. 关键技术点

### 3.1 悬浮球实现

**技术方案**: Tauri Window API

- **无边框窗口**: 使用`decorations: false`创建无边框窗口
- **始终置顶**: 使用`always_on_top: true`
- **圆形窗口**: 使用CSS `border-radius: 50%`实现圆形
- **拖拽**: 使用`draggable: true`或自定义拖拽逻辑

### 3.2 窗口管理

**技术方案**: Tauri Window API

- **展开/收起**: 通过改变窗口大小和位置实现
- **位置记忆**: 使用本地存储（SQLite或配置文件）保存窗口位置
- **多窗口**: 支持主窗口和悬浮球窗口

### 3.3 系统集成

**技术方案**: Tauri Plugins

- **系统托盘**: 使用`@tauri-apps/api`的tray API
- **开机自启动**: 使用`tauri-plugin-autostart`插件
  - Windows: 注册表方式
  - macOS: LaunchAgents方式
  - Linux: systemd/XDG Autostart方式
- **系统通知**: 使用`@tauri-apps/api`的notification API

### 3.4 自动点击功能

**技术方案**: 跨平台鼠标自动化库

**推荐库**:
- **enigo**: 跨平台鼠标键盘自动化库（支持Windows、macOS、Linux）
- **autopilot**: 跨平台GUI自动化库（功能更强大）

**权限要求**:
- Windows: 可能需要管理员权限
- macOS: 需要在系统偏好设置中授予辅助功能权限
- Linux: 可能需要X11权限

### 3.5 动画系统

**技术方案**: Framer Motion + Canvas/SVG

- **组件动画**: 使用Framer Motion处理组件动画（展开/收起、过渡等）
- **复杂图形**: 使用Canvas或SVG渲染复杂图形（如水晃动动画）
- **性能优化**: 
  - 使用`requestAnimationFrame`确保60fps
  - 使用GPU加速（CSS transform、Canvas）
  - 避免频繁重绘

### 3.6 数据持久化

**技术方案**: SQLite + Rust服务层

- **数据库初始化**: 应用启动时检查并初始化数据库
- **数据迁移**: 使用迁移脚本管理数据库结构变更
- **连接管理**: 使用连接池或单例模式管理数据库连接
- **错误处理**: 完善的错误处理和日志记录

## 6. 性能优化策略

### 6.1 内存优化

- **Rust后端**: Rust的内存安全保证，无需手动管理内存
- **前端GC**: React的自动垃圾回收
- **资源清理**: 及时清理事件监听器、定时器等
- **懒加载**: 按需加载资源

### 6.2 性能优化

- **动画优化**: 使用GPU加速，避免频繁重绘
- **数据库优化**: 使用索引，优化查询
- **渲染优化**: 使用虚拟列表（如果列表很长）
- **打包优化**: 代码分割，按需加载

### 6.3 启动优化

- **延迟加载**: 非关键功能延迟加载
- **资源优化**: 压缩资源，减少体积
- **数据库优化**: 数据库连接延迟初始化

## 7. 安全性考虑

### 7.1 数据安全

- **本地存储**: 所有数据本地存储，不上传到云端
- **数据加密**: 敏感数据可选择性加密（如果需要）

### 7.2 权限控制

- **最小权限**: 只请求必要的系统权限
- **权限说明**: 向用户说明权限用途

### 7.3 代码安全

- **依赖检查**: 定期检查依赖的安全漏洞
- **代码审查**: 代码审查确保安全性

## 8. 总结

本技术选型以**轻量化、跨平台、现代化**为核心原则，选择了成熟稳定的技术栈。Tauri + React + TypeScript的组合能够满足项目的所有需求，同时保证良好的性能和用户体验。未来功能的扩展（AI、皮肤等）也有相应的技术方案支持。
